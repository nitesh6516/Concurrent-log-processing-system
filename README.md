# Concurrent-log-processing-system

The problem asks to build a system that reads multiple log files from a directory, where each file contains user activity records in the format timestamp userId action. Since there are many files and sequential processing is too slow, you need to process each file in its own thread so they all run at the same time. Every thread parses its assigned file, skips any malformed or blank lines, and updates a shared data structure that counts how many actions each user performed. The challenge here is making sure multiple threads can safely write to that shared structure without corrupting the data.
Once all threads finish their work, you generate a final report file showing the aggregated user activity counts, sorted if possible. You also need to handle edge cases like missing directories, empty files, bad lines, and file read errors without crashing the program. The whole solution should be built using proper object-oriented design with separate classes for log entries, file processing, shared aggregation, and report generation.
